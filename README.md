# tree-sitter-duckyscript

A [Tree-sitter](https://tree-sitter.github.io/tree-sitter/) grammar for
[duckyPad duckyScript](https://github.com/duckyPad/duckyPad-Pro), the scripting
language used to program keys on the duckyPad macro-pad.

---

## Features

- Full syntax coverage of duckyScript 2 (duckyPad Pro firmware 2.x)
- Comments: `//`, `REM`, and `REM_BLOCK вҖҰ END_REM`
- String commands: `STRING`, `STRINGLN`, `STRING_BLOCK`, `STRINGLN_BLOCK`
- Variable interpolation with optional C-style format specifiers (`$var%10d`)
- OLED commands: `OLED_PRINT`, `OLED_CPRINT`, and all drawing commands
- Mouse commands: `MOUSE_MOVE`, `MOUSE_SCROLL`, click buttons
- LED commands: `SWC_SET`, `SWC_FILL`, `SWC_RESET`
- Profile commands: `GOTO_PROFILE`, `PREV_PROFILE`, `NEXT_PROFILE`
- Control flow: `IF / ELSE IF / ELSE / END_IF`, `WHILE / END_WHILE`
- Functions: `FUN вҖҰ END_FUN` / `FUNCTION вҖҰ END_FUNCTION`, `RETURN`
- Loop labels: `LOOP0:`, `LOOP1:`, вҖҰ
- Declarations: `DEFINE`, `VAR`
- All special / modifier / media / numpad key names
- All reserved variables (`_TIME_MS`, `_GV0`вҖ“`_GV31`, RTC vars, etc.)
- Full operator set: arithmetic, comparison, logical, bitwise, augmented assignment
- Syntax highlighting queries (`queries/highlights.scm`)
- Scope / locals queries (`queries/locals.scm`)
- Symbol-tag queries (`queries/tags.scm`)

## Building

### 1. Install dependencies

```sh
npm install
```

### 2. Generate the parser from `grammar.js`

```sh
npx tree-sitter generate
```

This writes the generated C source to `src/`. You must re-run this command
every time you edit `grammar.js`.

### 3. Compile the native Node.js binding (optional)

Only needed if you want to use the parser from Node.js directly.

```sh
node-gyp build
```

---

## Testing

The test corpus lives in `test/corpus/basics.txt`. Each entry contains a
snippet of duckyScript and the expected parse tree.

```sh
npx tree-sitter test
```

To run a single named test:

```sh
npx tree-sitter test -f "IF statement"
```

## Using with Neovim (nvim-treesitter)

1. Copy (or symlink) this directory to your nvim-treesitter `parsers` location,
   **or** add a custom parser entry to your config:

```lua
local parser_config = require("nvim-treesitter.parsers").get_parser_configs()
parser_config.duckyscript = {
  install_info = {
    url   = "/path/to/tree-sitter-duckyscript",
    files = { "src/parser.c" },
    generate_requires_npm   = false,
    requires_generate_from_grammar = true,
  },
  filetype = "ds",
}
```

2. Register the filetype:

```lua
vim.filetype.add({ extension = { ds = "duckyscript", dpds = "duckyscript" } })
```

3. Install the parser inside Neovim:

```
:TSInstall duckyscript
```

4. Copy the query files to your runtime queries directory so nvim-treesitter
   picks them up automatically:

```
~/.config/nvim/queries/duckyscript/highlights.scm
~/.config/nvim/queries/duckyscript/locals.scm
```

## Using with Helix

Add to `~/.config/helix/languages.toml`:

```toml
[[language]]
name        = "duckyscript"
scope       = "source.dpds"
file-types  = ["ds", "dpds"]
roots       = []
comment-token = "//"

[language.grammar]
name   = "duckyscript"
source = { path = "/path/to/tree-sitter-duckyscript" }
```

Then run `hx --grammar build` to compile.

## Grammar overview

### Node types

| Node | Description |
|------|-------------|
| `source_file` | Root node вҖ” the whole script |
| `comment` | `//` or `REM` line comment |
| `rem_block` | `REM_BLOCK вҖҰ END_REM` block comment |
| `string_block` | `STRING_BLOCK вҖҰ END_STRING` / `STRINGLN_BLOCK вҖҰ END_STRINGLN` |
| `string_command` | `STRING` / `STRINGLN` line |
| `oled_print_command` | `OLED_PRINT` / `OLED_CPRINT` line |
| `goto_profile_command` | `GOTO_PROFILE` line |
| `interpolation` | `$varname[%[width]type]` inside string content |
| `loop_label` | `LOOP0:` вҖҰ `LOOP9:` etc. |
| `if_statement` | Full `IF вҖҰ END_IF` block |
| `if_clause` | The opening `IF <cond>` line |
| `else_if_clause` | An `ELSE IF <cond>` branch |
| `else_clause` | An `ELSE` branch |
| `while_statement` | Full `WHILE вҖҰ END_WHILE` block |
| `function_def` | `FUN`/`FUNCTION` вҖҰ `END_FUN`/`END_FUNCTION` |
| `param_list` | Comma-separated parameter names |
| `block_body` | Body shared by if/while/function |
| `return_statement` | `RETURN [expr]` |
| `break_statement` | `LBREAK` |
| `continue_statement` | `CONTINUE` |
| `declaration` | `DEFINE` constant or `VAR` variable |
| `assignment` | `ident op expr` |
| `binary_expression` | Infix operator expression |
| `unary_expression` | Prefix `!`, `~`, or `-` |
| `function_call` | Call in expression context |
| `function_call_statement` | Call as a standalone statement |
| `key_press` | One or more key names on a line |
| `command` | All other built-in commands |
| `command_keyword` | The keyword token of a `command` |
| `reserved_variable` | `_TIME_MS`, `_GV0`, etc. |
| `number` | Decimal (`42`) or hexadecimal (`0xFF`) literal |
| `identifier` | User-defined name |

## Language reference

A quick summary of the duckyScript constructs that are parsed:

### Comments
```
// line comment
REM line comment
REM_BLOCK
  block comment (multiple lines)
END_REM
```

### Typing text
```
STRING  types this as-is
STRINGLN types this and presses Enter

STRING_BLOCK
everything here is typed verbatim
END_STRING

STRINGLN_BLOCK
each line types text then presses Enter
END_STRINGLN
```

### Variable interpolation
```
STRING value=$myVar
STRING hex=$myVar%x
STRING padded=$myVar%10d
STRING zero-padded=$myVar%010d
```

### Key presses
```
ENTER
CTRL ALT DELETE
WINDOWS r
SHIFT F10
```

### Timing
```
DELAY 500
DEFAULTDELAY 20
DEFAULTCHARDELAY 10
CHARJITTER 5
```

### Mouse
```
MOUSE_MOVE 10 -5
MOUSE_SCROLL 0 3
LMOUSE
RMOUSE
```

### OLED display
```
OLED_CURSOR 0 0
OLED_PRINT Hello!
OLED_CPRINT Centered
OLED_CIRCLE 64 32 10 0
OLED_LINE 0 0 127 63
OLED_RECT 0 0 64 32 1
OLED_CLEAR
OLED_UPDATE
OLED_RESTORE
```

### LED
```
SWC_SET 1 255 0 0
SWC_FILL 0 128 255
SWC_RESET 0
```

### Variables & constants
```
DEFINE MY_CONST 100
VAR x = 42
VAR h = 0xFF
x = x + 1
x *= 2
```

### Control flow
```
IF x == 0
  DELAY 100
ELSE IF x == 1
  DELAY 200
ELSE
  DELAY 300
END_IF

WHILE x > 0
  x -= 1
END_WHILE
```

### Functions
```
FUN greet(name)
  OLED_PRINT Hello $name
  RETURN 0
END_FUN

greet(myVar)
```

### Loop labels
```
LOOP0:
STRING action one

LOOP1:
STRING action two
```

### Reserved variables (read/write)
```
VAR t  = _TIME_MS
VAR nl = _IS_NUMLOCK_ON
_CHARJITTER = 10
_GV0 = 42           // persistent global
```
